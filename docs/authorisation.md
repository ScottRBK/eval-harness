# Authorisation 

Authorisation is currently accessible via subscriptions, this is managed through various means,
either by mounting the auth files of an agent, or if possible using an oauth token that can be
generated by your coding agent, such as using claude codes `claude setup-token`. The below matrix
depicts how each harness manages this. 

I will eventually also simply add API tokens as well, for the ballers amongst us who can afford to 
do that. 

Every agent here authenticates against a **subscription**. Settings are read from `.env` and take
the `EVAL_HARNESS_` prefix (so the Codex row's setting is `EVAL_HARNESS_CODEX_CREDENTIALS_LOC`).

| Agent       | Method             | How to obtain         | Harness setting           |
| ----------- | ------------------ | --------------------- | ------------------------- |
| Claude Code | OAuth token (env)  | `claude setup-token`  | `CLAUDE_CODE_OAUTH_TOKEN` |
| OpenCode    | Mounted auth file  | `opencode auth login` | `OPENCODE_CREDENTIALS_LOC`|
| Codex       | Mounted auth file  | `codex login`         | `CODEX_CREDENTIALS_LOC`   |
| Copilot CLI | GitHub token (env) | fine-grained PAT[^1]  | `COPILOT_GITHUB_TOKEN`    |
| Pi | _not implemented_  | —                     | —                         |

**Mounted auth file** — the harness stages a throwaway copy of the host auth file and bind-mounts it
into the container (OpenCode to `/home/node/.local/share/opencode`, Codex to `/home/node/.codex`).
The agent may refresh the token in place during the run; the copy is discarded afterwards, so the
host file is never touched. The `*_CREDENTIALS_LOC` setting is a path and defaults to the agent's
standard location, so you usually need not set it.

**Token (env)** — the harness injects the credential as a container environment variable. Claude
Code's `setup-token` mints a long-lived OAuth token; for Copilot you supply a GitHub token.

A missing credential fails fast at provisioning with a clear error before any container starts.

[^1]: A **user-owned** fine-grained PAT carrying the **Copilot Requests** permission (classic `ghp_`
tokens are rejected). `gh auth token` may work if its token carries that permission.
